# Home Assistant Integration Roadmap for EMHASS

This document analyzes the current state of EMHASS integration with Home Assistant and proposes a roadmap for making it more seamless and user-friendly.

## Current Integration Analysis

### Current State: API-Driven Architecture

EMHASS currently integrates with Home Assistant through:

1. **REST API Calls**: Manual curl commands for optimization triggers
2. **Data Retrieval**: HTTP requests to HA history API for sensor data
3. **Data Publishing**: HTTP POST to HA states API for result sensors
4. **Add-on Deployment**: Docker container with supervisor API access
5. **Manual Configuration**: YAML/JSON configuration files

### Current User Experience Pain Points

1. **Complex Setup**: Users must understand REST APIs, configure sensors, and write automations
2. **Manual Triggers**: Optimization requires explicit automation setup
3. **Error Prone**: URL/token management, JSON syntax, API endpoints
4. **Limited Discovery**: No automatic detection of suitable sensors
5. **Maintenance Overhead**: Updates require manual configuration changes
6. **Disconnected UI**: Web interface separate from Home Assistant dashboard

```python
class SensorDiscovery:
    def discover_power_sensors(self):
        """Auto-discover power-related sensors"""
        candidates = {
            'load_sensors': [],
            'pv_sensors': [],
            'battery_sensors': [],
            'grid_sensors': []
        }
        
        # Query HA for sensors with power device_class
        sensors = self.get_all_sensors()
        for sensor in sensors:
            if sensor.device_class == 'power':
                # Classify by name patterns
                if any(x in sensor.name.lower() for x in ['load', 'consumption', 'usage']):
                    candidates['load_sensors'].append(sensor)
                elif any(x in sensor.name.lower() for x in ['solar', 'pv', 'photovoltaic']):
                    candidates['pv_sensors'].append(sensor)
                # ... etc
        
        return candidates

    def suggest_configuration(self):
        """Generate configuration suggestions"""
        sensors = self.discover_power_sensors()
        return {
            'sensor_power_load_no_var_loads': sensors['load_sensors'][0] if sensors['load_sensors'] else None,
            'sensor_power_photovoltaics': sensors['pv_sensors'][0] if sensors['pv_sensors'] else None,
            # ... automated suggestions
        }
```

### Phase 2: Advanced Add-on Features (Medium Term - 6-12 months)

#### 2.1 Device Integration
**Goal**: Automatic device control without manual automations

```python
# New module: src/emhass/device_controller.py
class DeviceController:
    def __init__(self, ha_client):
        self.ha = ha_client
        self.controlled_devices = {}
    
    def register_device(self, emhass_sensor, ha_switch):
        """Link EMHASS optimization output to HA device"""
        self.controlled_devices[emhass_sensor] = {
            'switch': ha_switch,
            'last_state': None,
            'control_method': 'binary'  # or 'continuous', 'scheduled'
        }
    
    def update_devices(self):
        """Update device states based on optimization results"""
        for sensor, config in self.controlled_devices.items():
            current_value = self.ha.get_sensor_value(sensor)
            
            if config['control_method'] == 'binary':
                should_be_on = current_value > 0.1
                if should_be_on != config['last_state']:
                    self.ha.call_service(
                        'switch.turn_on' if should_be_on else 'switch.turn_off',
                        entity_id=config['switch']
                    )
                    config['last_state'] = should_be_on
```

```python
class EMHASSConfigFlow(config_entries.ConfigFlow, domain=DOMAIN):
    VERSION = 1
    
    async def async_step_user(self, user_input=None):
        """Handle initial setup step"""
        if user_input is not None:
            return await self.async_step_sensors()
            
        return self.async_show_form(
            step_id="user",
            data_schema=vol.Schema({
                vol.Required("server_url", default="http://localhost:5000"): str,
                vol.Optional("has_solar", default=False): bool,
                vol.Optional("has_battery", default=False): bool,
            })
        )
    
    async def async_step_sensors(self, user_input=None):
        """Sensor selection step"""
        available_sensors = await self._discover_sensors()
        
        return self.async_show_form(
            step_id="sensors",
            data_schema=vol.Schema({
                vol.Required("load_sensor"): vol.In(available_sensors["power"]),
                vol.Optional("pv_sensor"): vol.In(available_sensors["power"]),
            })
        )
```

#### 3.3 Data Coordinator
**Goal**: Efficient data management and updates

```python
# custom_components/emhass/coordinator.py
class EMHASSDataCoordinator(DataUpdateCoordinator):
    def __init__(self, hass, config):
        super().__init__(
            hass,
            _LOGGER,
            name=DOMAIN,
            update_interval=timedelta(minutes=5),
        )
        self.emhass_client = EMHASSClient(config)
    
    async def _async_update_data(self):
        """Fetch data from EMHASS"""
        try:
            # Get current optimization results
            data = await self.emhass_client.get_current_data()
            
            # Auto-trigger optimization if needed
            if self._should_run_optimization():
                await self.emhass_client.run_dayahead_optimization()
                data = await self.emhass_client.get_current_data()
            
            return data
        except Exception as ex:
            raise UpdateFailed(f"Error communicating with EMHASS: {ex}")
```

## Implementation Strategy

### Recommended Approach: Incremental Evolution

**Phase 1 (Immediate - Next Release)**
- Sensor discovery and setup wizard in existing add-on
- Improved web UI with guided setup
- Better error handling and user feedback

**Phase 2 (Medium Term)**
- Enhanced integration with popular HA components
- Device control automation
- Dashboard card development

**Phase 3 (Long Term)**
- Full custom component when user base and stability warrant it
- HACS integration for easy installation
- Advanced features like scene integration, blueprint automation

### User Experience Goals

#### For New Users:
1. **One-Click Setup**: Automatic sensor discovery and configuration
2. **Guided Configuration**: Step-by-step wizard with explanations
3. **Automatic Device Linking**: Direct control without manual automations

#### For Advanced Users:
1. **Full API Access**: Maintain current flexibility
2. **Custom Blueprints**: Pre-built automation templates
3. **Advanced Configuration**: Deep customization options

#### For All Users:
1. **Native HA Feel**: Integrates seamlessly with HA UI
2. **Reliable Operation**: Robust error handling and recovery
3. **Clear Feedback**: Status indicators and helpful error messages

## Technical Implementation Notes

### Backward Compatibility
- Maintain current REST API for existing users
- Gradual migration path for configurations
- Support both old and new integration methods during transition

### Dependencies
- Minimize additional dependencies for custom component
- Use HA's built-in HTTP client and data coordinators
- Leverage existing HA infrastructure (config flow, etc.)

### Performance Considerations
- Cache frequently accessed data
- Async operations for all HA communication
- Configurable update intervals
- Smart optimization triggering

### Security & Reliability
- Use HA's built-in authentication
- Proper error handling and retry logic
- Configuration validation
- Graceful degradation when EMHASS service unavailable

## Conclusion

The roadmap provides a clear path from the current API-driven integration to a fully native Home Assistant experience. The incremental approach allows for:

1. **Immediate improvements** that enhance the current user experience
2. **Medium-term features** that add significant value without major architectural changes  
3. **Long-term vision** of a fully integrated custom component

This approach balances user needs, development resources, and technical complexity while maintaining the powerful optimization capabilities that make EMHASS valuable.
